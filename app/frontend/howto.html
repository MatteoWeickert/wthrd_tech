How-To-PyStac

Einführung

Ein STAC (SpatioTemporal Asset Catalog) bietet eine standardisierte Möglichkeit, Geodaten zu katalogisieren und darauf zuzugreifen. Mit der Python-Bibliothek PySTAC kann diese Struktur leicht durchsucht und verarbeitet werden. Hier erklären wir Schritt für Schritt, wie man mit PySTAC arbeitet.

Schritt-für-Schritt-Erklärung

Verbindung zu einem STAC-Katalog herstellen

Die Route zum STAC wird über die url festgelegt. Mit Catalog.from_file(url) wird die Verbindung zur STAC-API hergestellt.

url = "http://localhost:8000"
try:
    catalog = Catalog.from_file(url)
    print("Katalog erfolgreich geladen")
except Exception as e:
    print(f"Fehler beim Laden des Katalogs: {e}")

Konformität des STAC abrufen

Die Konformitätsklassen, mit denen der STAC übereinstimmt, lassen sich über die /conformance-Route abfragen:

try:
    response = requests.get(f"{url}/conformance")
    if response.status_code == 200:
        conformance_info = response.json()
        print("API Conformance Informationen:")
        print(json.dumps(conformance_info, indent=4))
    else:
        print(f"Fehler beim Abrufen der Conformance-Daten: {response.status_code}")
except Exception as e:
    print(f"Fehler beim Abrufen der Conformance-Informationen: {e}")

Collections und ihre Items abrufen

Mit collection.describe() lassen sich die Item-IDs der Collections abrufen. Hier werden alle Collections des Katalogs mit ihren Items abgefragt:

try:
    collections = catalog.get_collections()
    print("Sammlungen im Katalog:")
    for collection in collections:
        collection.describe()
except Exception as e:
    print(f"Fehler beim Abrufen der Collections: {e}")

Eine spezifische Collection abrufen

Collections lassen sich anhand ihrer ID abrufen. Hier holen wir eine Collection mit der ID "MLM_Collection":

collection_id = "MLM_Collection"
try:
    collection = catalog.get_child(collection_id)
    print(f"Collection mit ID {collection_id} gefunden.")
    collection.describe()
except Exception as e:
    print(f"Fehler beim Abrufen der Collection {collection_id}: {e}")

Items aus der Collection abrufen

Nach dem Abrufen einer Collection können Sie alle enthaltenen Items durchsuchen. Hier werden bereits die wichtigsten Attribute Selektiert und in die Konsole ausgegeben:

collection_id = "MLM_Collection"
try:
    collection = catalog.get_child(collection_id)
    items = collection.get_all_items()
    for item in items:
        assets = {key: asset.to_dict() for key, asset in item.assets.items()}
        item_summary = {
            "ID": item.id,
            "Bounding Box": item.bbox,
            "Datetime": item.datetime.isoformat() if item.datetime else None,
            "Properties": item.properties,
            "Assets": assets,
        }
        print(json.dumps(item_summary, indent=4) )
except Exception as e:
    print(f"Fehler beim Abrufen der Items: {e}")

Suche nach bestimmten Items

Als erstes definiert man die Suchparameter. Dabei stehen eine Boundingbox, Collection-Namen, ein bestimmtes Zeitintervall und Item-IDs als Parameter zur Verfügung. Außerdem lässt sich die Suche auf eine bestimmte Anzahl an Ergebnissen limitieren:

bbox = [-180, -90, 180, 90]
datetime = "2020-01-01T00:00:00/2025-01-15T23:59:59"
collections = ["MLM_Collection"]
ids = ["solar_satlas_sentinel2"]
limit = 10

Die Suchanfrage wird dann an den STAC-Server gesendet und die Ergebnisse werden in der Konsole ausgegeben:

search_query = {
    "bbox": bbox,
    "collections": collections,
    "datetime": datetime,
    "ids": ids,
    "limit": limit
}
try:
    response = requests.post(f"{url}/search", json=search_query)
    if response.status_code == 200:
        items = response.json().get('features', [])
        print(json.dumps(items, indent=4))
        print(f"Gefundene Items: {len(items)}")
    else:
        print(f"Fehler bei der Suche: {response.status_code} - {response.text}")
except Exception as e:
    print(f"Fehler bei der Suche nach Items: {e}")

Assets eines Items abrufen

Nachdem Sie ein spezifisches Item gefunden haben, können Sie dessen Assets anzeigen:

item_id = "solar_satlas_sentinel2"
items = catalog.get_all_items()
for item in items:
    if item.id == item_id:
        print(f"Assets für Item {item_id}:")
        for key, asset in item.assets.items():
            print(f"Asset: {key}")
            print(json.dumps(asset.to_dict(), indent=4))

Zusammenfassung

Mit PySTAC wird der Zugriff auf STAC-Kataloge, das Abrufen von Collections und Items sowie das Finden spezifischer Daten vereinfacht. Der Code kann flexibel angepasst werden, um die Möglichkeiten des Katalogs optimal zu nutzen.

How-To-Katalog

Modell hinzufügen

Fügen Sie hier ein neues Modell hinzu, um Ihre Daten für die Analyse und Verarbeitung vorzubereiten. Wählen Sie aus einer Vielzahl von vorgefertigten Modellen oder laden Sie Ihre eigenen hoch, um maßgeschneiderte Ergebnisse zu erzielen. Dieses Schritt-für-Schritt-Verfahren führt Sie durch den gesamten Prozess, von der Auswahl des Modells bis hin zur Integration in Ihre Workflows.

1. Einführung

Bitte beachten Sie, dass Sie, um ein eigenes Modell im wthrd.tech-Katalog hinzufügen zu können, über einen Account verfügen müssen. Registrieren Sie sich kostenlos und melden Sie sich anschließend an.

2. Benötigte Parameter

Für das Hinzufügen eines Modells sind einige Parameter notwendig.

  • Name: Wählen Sie hier einen aussagekräftigen Namen evtl. mit Version
  • Tasks: Wählen Sie hier aus dem Pool an bereitgestellten Tasks die zutreffenden aus. Tipp: Wählen Sie so viele wie möglich.
  • Description: Beschreiben Sie hier Ihr Modell umfassend, sodass fremde Nutzer Ihr Modell ohne weiteres verstehen kann.
  • ID: Wähle hier eine eindeutige Modell-ID. Sollte diese schon belegt sein, wird Ihnen der Katalog dies mitteilen.
  • Description: Beschreiben Sie hier eine Geometrie in folgender Form: 
    
{ 
    "type": 
    "Polygon", 
    "coordinates": [ 
    [ 
    [ -7.882190080512502, 37.13739173208318 ], 
    [ -7.882190080512502, 58.21798141355221 ], 
    [ 27.911651652899923, 58.21798141355221 ], 
    [ 27.911651652899923, 37.13739173208318 ], 
    [ -7.882190080512502, 37.13739173208318 ] 
    ] ] 
}
  • Inputname: Wählen Sie hier die erwartete Eingabe, die Ihr Modell erwartet. Bleiben Sie dabei zunächst oberflächig, und beschreiben den InputName z.B. 'Image' oder mit 'Audio'.
  • Inputtypes: Beschreiben Sie an dieser Stelle, welche Dateitypen Ihr Modell akzeptiert. Wenn Sie mehrere hinzufügen möchten, trennen Sie Ihre Eingaben mit einem Komma (,).
  • Architecture: Wählen Sie hier, nach welcher Architektur Ihr Modell erstellt wurde.
  • Framework: Beschreiben Sie, welches Framework Ihr Modell nutzt.
  • Framework-Version Geben Sie an, welche Version das von Ihnen genutzte Framework zum Zeitpunkt der Erstellung nutzt
  • Accelerator: Wählen Sie, welche Art von Accelerator für Ihr Modell genutzt wurde.
  • Acceleratorsummary: Geben Sie hier wenn notwendig weitere Informationen zu dem angegebenen Accelerator an. Sinnvolle Informationen wären an dieser Stelle, wenn nur bestimmte Arten CPUs benutzt wurden.
  • Pretrained-Source: Wählen Sie hier, aus welcher Quelle Sie hauptsächlich Ihre Daten für das Training des Modells bezogen haben.
  • Batchsize-Suggestion: Geben Sie an, mit wie vielen Trainingsdaten Ihr Modell im Optimalfall benötigt.
  • Hyperparameter: Geben Sie hier weitere Informationen zu Ihrem Modell an. Anders als bei den vorherigen Parametern können die Key-Value-Pairs frei belegt und genutzt werden. Das Beispiel im Folgenden ist daher keine Vorage. 
    
{            
    "learning_rate": 0.001,             
    "dropout": 0.5         
}
  • CollectionID: Wählen Sie hier aus den vorgegebenen Collections eine aus, in welcher Sie Ihr Modell hinterlegen möchten. Im Folgenden wird ebenfalls erläutert, wie Sie eine eigene Collection anlegen können.
  • Link: Nutzen Sie dieses Feld nur für den direkten Link zu Ihrem Modell.
  • Bounding-Box: Zeichnen Sie das Gebiet ein, auf das Ihr Modell trainiert wurde bzw. auf das Ihr Modell anwendbar ist.
  • Farbgebung: Wählen Sie eine Farbe, passend zu Ihrem Modell. Ihr Modell wird dann in dieser Farbe im wthrd.tech-Katalog angezeigt.
  • Zeitraum: Wählen Sie, für welchen Zeitraum Ihr Modell trainiert wurde.
  • Vortrainiert: Geben Sie an, ob Ihr Modell anhand von (Traings-)daten vortrainiert wurde.

Collection hinzufügen

In diesem Abschnitt können Sie Ihre Sammlung von Modellen, Daten oder Ressourcen verwalten. Eine Sammlung hilft Ihnen dabei, alle relevanten Elemente in einer übersichtlichen und leicht zugänglichen Struktur zu organisieren. Sie können Ihre Sammlung jederzeit erweitern, indem Sie neue Elemente hinzufügen, um Ihre Projekte effizient zu verwalten.

1. Einführung

Bitte beachten Sie, dass Sie, um eine eigene Collection im wthrd.tech-Katalog hinzufügen zu können, über einen Account verfügen müssen. Registrieren Sie sich kostenlos und melden Sie sich anschließend an.

2. Benötigte Parameter

Für das Hinzufügen einer Collection sind einige Parameter notwendig.

  • ID:Wähle hier eine eindeutige Collection-ID. Sollte diese schon belegt sein, wird Ihnen der Katalog dies mitteilen.
  • Title: Wählen Sie einen aussagekräftigen Titel, der unteranderem den räumlichen Bezug der Collection bzw. der Einschränkung beschreibt.
  • Description: Beschreiben Sie Ihre Collection ausführlich hinsichtlich der räumlichen Einschränkung und angestrebten Modelle.
  • License: Geben Sie hier die Lizenz an.
  • Bounding-Box: Zeichnen Sie, für welches Gebiet Ihre Collection Modelle halten soll. Beachten Sie, dass beim hinzufügen eines Modells in eine Collection keine Überprüfung der Bounding-Boxen stattfindet.
  • Zeitraum: Wählen Sie einen Zeitraum, für welches Ihre Collection Modelle halten soll.

Verfügbare Aufgaben

Hier finden Sie eine Übersicht aller verfügbaren Aufgaben, die im Rahmen Ihres Modells oder Ihrer Collection bearbeitet werden können. Jede Aufgabe ist spezifisch auf bestimmte Anforderungen ausgelegt.

  • Any-to-Any: Eine Methode zur Modellierung von Beziehungen zwischen beliebigen Eingabe- und Ausgabedaten, wobei das Modell für jedes beliebige Paar von Eingabe- und Ausgabedaten trainiert wird.
  • Audio Classification: Klassifikation von Audiodaten in verschiedene Kategorien basierend auf ihrer akustischen Merkmale, z. B. Musikgenres oder gesprochene Sprache.
  • Audio-Text-to-Text: Die Umwandlung von Audioinhalten in Text und die anschließende Textverarbeitung, wie etwa Textklassifikation oder Zusammenfassung.
  • Audio-to-Audio: Die Umwandlung von Audiodaten von einer Quelle in eine andere, z. B. von Rauschen zu klarem Audio oder von einer Sprache in eine andere.
  • Automatic Speech Recognition: Die automatische Umwandlung von gesprochener Sprache in Text, die oft für Transkriptionen und Sprachsteuerung verwendet wird.
  • Depth Estimation: Die Schätzung der Tiefe in Bildern oder Videos, um dreidimensionale Informationen aus zweidimensionalen Daten zu extrahieren.
  • Document Question Answering: Die Fähigkeit eines Modells, Fragen zu einem Dokument zu verstehen und relevante Antworten aus dem Text zu extrahieren.
  • Feature Extraction: Der Prozess der Extraktion relevanter Merkmale oder Eigenschaften aus Rohdaten, um sie für eine Modellierung oder Analyse nutzbar zu machen.
  • Fill-Mask: Ein Verfahren zur Maskierung bestimmter Bereiche eines Textes oder Bildes, das Modell wird dann trainiert, diese maskierten Bereiche zu füllen oder vorherzusagen.
  • Image Classification: Die Klassifikation von Bildern in vordefinierte Kategorien basierend auf den visuellen Inhalten des Bildes.
  • Image Feature Extraction: Extraktion von Merkmalen aus Bildern, um diese für weiterführende Modelle oder Analysen vorzubereiten, wie z. B. für die Bildklassifikation oder -analyse.
  • Image Segmentation: Die Aufteilung eines Bildes in verschiedene Bereiche, um Objekte oder Szenen zu isolieren und zu analysieren.
  • Image-Text-to-Text: Die Transformation von Bildinhalten in Text, der dann weiter verarbeitet werden kann, z. B. durch Beschreibung oder Klassifikation.
  • Image-to-3D: Die Umwandlung von zweidimensionalen Bildern in dreidimensionale Modelle, oft verwendet in der Computergrafik oder für virtuelle Realitäten.
  • Image-to-Image: Die Umwandlung eines Bildes in ein anderes Bild, wobei das Modell zwischen verschiedenen Bildstilen oder -arten übersetzt, z. B. von Skizzen zu realistischen Bildern.
  • Image-to-Text: Die Umwandlung von Bildinhalten in beschreibenden Text, häufig verwendet in der Bildunterschriften-Generierung oder für visuelle Fragenbeantwortung.
  • Keypoint Detection: Die Erkennung und Lokalisation spezifischer Merkmale oder "Schlüsselpunkte" in einem Bild, die für Aufgaben wie Objektverfolgung oder Gesichtserkennung verwendet werden.
  • Msaak Generation: Eine Methode zur Erstellung von musikspezifischen Sequenzen oder Audioabschnitten, die auf bestimmten Eingabedaten basieren.
  • Object Detection: Die Identifizierung und Klassifikation von Objekten in Bildern oder Videos, einschließlich der Bestimmung ihrer Position und Größe.
  • Question Answering: Modelle, die Fragen zu einem Text verstehen und präzise Antworten aus den gegebenen Informationen extrahieren.
  • Reinforcement Learning: Ein Lernprozess, bei dem ein Modell durch Belohnungen oder Strafen lernt, um in einer gegebenen Umgebung optimale Entscheidungen zu treffen.
  • Sentence Similarity: Die Bestimmung der Ähnlichkeit zwischen zwei Sätzen, basierend auf ihrem Inhalt, was häufig in Suchmaschinen oder Übersetzungsmodellen verwendet wird.
  • Summarization: Die automatische Erstellung einer kürzeren Version eines Textes, die die wesentlichen Informationen beibehält, häufig genutzt für Nachrichten- oder Inhaltszusammenfassungen.
  • Table Question Answering: Die Fähigkeit eines Modells, Fragen zu Tabellen zu beantworten, basierend auf den Daten, die in den Zellen enthalten sind.
  • Tabular Classification: Die Klassifikation von Daten in Tabellenformat, um Vorhersagen oder Gruppierungen auf Basis von Attributen vorzunehmen.
  • Tabular Regression: Die Vorhersage eines kontinuierlichen Werts aus tabellarischen Daten, häufig in wirtschaftlichen oder wissenschaftlichen Anwendungsbereichen.
  • Text Classification: Die Zuweisung von Texten zu vordefinierten Kategorien, z. B. zur Spam-Erkennung oder Sentiment-Analyse.
  • Text Generation: Die automatische Erstellung von Texten, basierend auf einer Eingabe oder einem bestimmten Thema, häufig verwendet in Chatbots oder Content-Generierung.
  • Text-to-3D: Die Erstellung von 3D-Modellen oder Szenen basierend auf textuellen Beschreibungen, was oft in virtuellen Umgebungen oder Designprozessen verwendet wird.
  • Text-to-Image: Die Umwandlung von Textbeschreibungen in Bilder, basierend auf den im Text angegebenen Attributen.
  • Text-to-Speech: Die Umwandlung von Text in gesprochene Sprache, um eine sprachgesteuerte Interaktion mit Maschinen zu ermöglichen.
  • Text-to-Video: Die Umwandlung von Textinformationen in ein Videoformat, das visuelle und auditive Elemente enthält.
  • Token Classification: Die Klassifikation von Tokens (z. B. Wörtern oder Satzzeichen) in Texten, häufig verwendet in Named-Entity-Recognition oder anderen NLP-Aufgaben.
  • Translation: Die Umwandlung von Texten aus einer Sprache in eine andere, eine grundlegende Fähigkeit in der maschinellen Übersetzung.
  • Unconditional Image Generation: Die generative Erstellung von Bildern ohne Einschränkungen oder Bedingungen, oft verwendet in der Kunst oder kreativen Prozessen.
  • Video Classification: Die Klassifikation von Videos in Kategorien basierend auf ihrem Inhalt, wie z. B. Erkennung von Aktivitäten oder Szenen in einem Video.
  • Video-Text-to-Text: Die Umwandlung von Videoinhalten in Text, um wichtige Informationen aus dem Video zu extrahieren und für Textverarbeitung zu nutzen.
  • Visual Question Answering: Eine Kombination von Bildverarbeitung und NLP, bei der das Modell auf visuelle Fragen eine präzise Antwort basierend auf Bildinhalten gibt.
  • Zero-Shot Classification: Die Fähigkeit eines Modells, Daten zu klassifizieren, für die es nicht explizit trainiert wurde, indem es auf allgemeines Wissen und Erfahrung zugreift.
  • Zero-Shot Image Classification: Klassifikation von Bildern in Kategorien, für die das Modell nicht speziell trainiert wurde, häufig durch die Verwendung von semantischen Embeddings.
  • Zero-Shot Object Detection: Die Fähigkeit, Objekte zu erkennen, ohne dass das Modell zuvor für diese spezifischen Objekte trainiert wurde, basierend auf allgemeinen Konzepten und Features.
